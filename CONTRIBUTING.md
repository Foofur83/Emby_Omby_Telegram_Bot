# Contributing to Emby/Ombi Telegram Bot

Bedankt voor je interesse om bij te dragen aan dit project! 🎉

## Hoe kun je bijdragen?

### 🐛 Bug Reports

Als je een bug vindt, open dan een [GitHub Issue](https://github.com/Foofur83/Emby_Omby_Telegram_Bot/issues) met:

- **Duidelijke titel** die het probleem beschrijft
- **Stappen om te reproduceren** - hoe kunnen we de bug zien?
- **Verwacht gedrag** - wat zou er moeten gebeuren?
- **Actueel gedrag** - wat gebeurt er nu?
- **Log output** - relevante error messages (zonder API keys!)
- **Omgeving** - Python versie, OS, Docker/local deployment

### 💡 Feature Requests

Heb je een idee voor een nieuwe functie? Open een Issue met:

- **Beschrijving** van de feature
- **Use case** - waarom is dit nuttig?
- **Voorbeelden** - hoe zou het werken?

### 🔧 Pull Requests

Wil je zelf code bijdragen? Geweldig!

1. **Fork** de repository
2. **Clone** je fork lokaal
3. **Maak een branch** voor je feature: `git checkout -b feature/mijn-feature`
4. **Maak je changes** en test ze grondig
5. **Commit** met duidelijke messages: `git commit -m "Add: nieuwe feature beschrijving"`
6. **Push** naar je fork: `git push origin feature/mijn-feature`
7. **Open een Pull Request** naar `main` branch

### 📝 Code Style

- Volg bestaande code formatting
- Gebruik duidelijke variabele namen (Nederlands of Engels)
- Voeg comments toe voor complexe logica
- Update documentatie bij nieuwe features

### ✅ Testing

Voor het indienen van een PR, test:

- [ ] Bot start zonder errors
- [ ] Nieuwe functionaliteit werkt zoals verwacht
- [ ] Bestaande functionaliteit is niet gebroken
- [ ] Web interface werkt (indien aangepast)
- [ ] Config voorbeeld is up-to-date

### 🔒 Veiligheid

- **NOOIT** API keys, tokens of wachtwoorden committen
- Test met `config.yaml.example` dat geen echte credentials nodig zijn
- Voeg gevoelige files toe aan `.gitignore`

## Vragen?

Open een Issue of discussion op GitHub!
